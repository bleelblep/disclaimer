Custom Packs Guide

Learn how to create distributable Pixling packs with automatic update checking, version management, and professional hosting.

📦

What are Custom Packs?

Custom Packs are collections of Pixlings bundled with metadata like author information, version numbers, and update URLs. Unlike individual custom Pixlings, packs support automatic update checking, making distribution and maintenance much easier.

Before You Start
Make sure you're familiar with creating custom Pixlings first. Check out the Custom Pixlings Guide to learn the basics of animation creation.
📁

Pack Structure

A custom pack is a ZIP file with the following structure:

Directory Structure
my-pack.zip
├── pack.json                 (required - pack metadata)
└── animations/               (required - animation files)
    ├── MyPixling_Idle.json
    ├── MyPixling_Motion.json
    └── ...

The pack.json file contains all metadata about your pack and its Pixlings.

📝

pack.json Format

Here's a complete example of a pack.json file with all fields:

pack.json
{
  "name": "My Custom Pack",
  "subtitle": "A collection of awesome Pixlings",
  "author": "Your Name",
  "version": "1.0.0",
  "description": "A detailed description of your pack and what makes it special.",
  "created": "2026-03-31",
  "updateUrl": "https://github.com/yourusername/my-pack/releases/latest/download/pack.json",
  "pixlings": [
    {
      "name": "My Pixling",
      "subtitle": "A friendly companion",
      "description": "This Pixling loves to dance and react to notifications.",
      "defaultIdleAnimation": "MyPixling_Idle.json",
      "animations": {
        "Idle": ["MyPixling_Idle.json"],
        "Motion": ["MyPixling_Motion.json"],
        "Notification": [
          "MyPixling_Notification_Enter.json",
          "MyPixling_Notification_Loop.json",
          "MyPixling_Notification_Exit.json"
        ]
      }
    }
  ]
}
Field Required Description
name ✅ Yes Pack name (shown in the app)
subtitle ✅ Yes Short description
author Recommended Creator name or organization
version Recommended Pack version (e.g., "1.0.0")
description Optional Detailed pack description
created Optional Creation date (any format)
updateUrl Optional URL to check for updates (see below)
pixlings ✅ Yes Array of Pixling definitions
🔄

Automatic Update Checking

The updateUrl field enables automatic update checking. When set, Pixlings will periodically check this URL for a newer version of your pack.

How It Works
The app downloads the pack.json from your updateUrl and compares the version field. If the remote version is newer, users will see an "Update Available" notification.

Important: The updateUrl should point to a pack.json file, NOT the ZIP file. Users will download the full ZIP when they choose to update.

Example updateUrl
"updateUrl": "https://github.com/user/repo/releases/latest/download/pack.json"
🏷️

Versioning Best Practices

Use semantic versioning (MAJOR.MINOR.PATCH) for clear version tracking:

  • 1.0.0 → Initial release
  • 1.0.1 → Bug fix or small tweak
  • 1.1.0 → New Pixling or feature added
  • 2.0.0 → Major redesign or breaking change

The app uses smart version comparison:

  • ✅ Semantic versions are compared numerically (1.0.0 < 1.0.1)
  • ✅ Non-semantic versions use string comparison
  • ✅ Missing version numbers default to 0 (1.0 = 1.0.0)
🌐

Hosting Your Pack

You need to host both your pack ZIP file and a publicly accessible pack.json for update checking. Here are the recommended options:

GitHub Releases (Recommended)

The easiest and most reliable option for hosting packs.

  1. Create a GitHub repository for your pack
  2. In your repo, go to Releases → Create a new release
  3. Tag the release with your version (e.g., v1.0.0)
  4. Upload two files as release assets:
    • my-pack.zip - The full pack
    • pack.json - Just the metadata file
  5. In your ZIP's pack.json, set the updateUrl to: 
    "updateUrl": "https://github.com/USER/REPO/releases/latest/download/pack.json"
  6. Share the ZIP download link with users: 
    https://github.com/USER/REPO/releases/latest/download/my-pack.zip
Pro Tip
Using /releases/latest/download/ means users always get the newest version without you updating any links. Just create a new release and it works automatically!

GitHub Pages

Good for custom landing pages and documentation.

  1. In your repo settings, enable GitHub Pages
  2. Commit your pack.json to the docs/ folder or root
  3. Use the GitHub Pages URL as your updateUrl: 
    "updateUrl": "https://USER.github.io/REPO/pack.json"
  4. Host the ZIP file via GitHub Releases (recommended) or in your repo

Other Hosting Options

  • Dropbox/Google Drive: Generate public share links, but these may change or expire
  • Your own website: Host both files on your server with direct HTTPS links
  • Pastebin/Gist: For pack.json only (use raw URL), but not for ZIPs
🧪

Testing Your Pack

  1. Create your pack.json and animation files
  2. ZIP them together (not in a subfolder!)
  3. Import the ZIP into Pixlings via the Terrarium screen
  4. Verify all Pixlings appear correctly
  5. Test each animation state (Idle, Motion, Notification, etc.)
  6. If everything works, upload to your hosting platform
  7. Test the update URL by importing the pack, then creating a new version with a higher version number and checking for updates in the Pack Details screen
📢

Distributing Your Pack

Once your pack is hosted and tested, you can share it with others:

  • Share the direct ZIP download link
  • Users can tap the link on their phone to download
  • After importing, they'll automatically receive update notifications
  • Consider creating a README or landing page with screenshots and installation instructions
Community Sharing
Consider sharing your pack on Reddit, Discord, or other Nothing Phone communities. Always include clear installation instructions and screenshots!
⚡

Quick Start Workflow

  1. Create your Pixling animations
  2. Write a pack.json with metadata and version 1.0.0
  3. ZIP the files together
  4. Create a GitHub repository
  5. Create a release tagged v1.0.0
  6. Upload my-pack.zip and pack.json as release assets
  7. Set updateUrl to https://github.com/USER/REPO/releases/latest/download/pack.json
  8. Re-ZIP with the updated pack.json
  9. Replace the ZIP in the release
  10. Share the download link!

When you want to release an update:

  1. Update your animations or pack.json
  2. Bump the version number (e.g., 1.0.0 → 1.1.0)
  3. Create a new GitHub release with the new version tag
  4. Upload the new ZIP and pack.json
  5. Users will automatically see "Update Available" in the app!
💡

Pro Tips

  • Use descriptive names: "Cyberpunk Companions v2" is better than "My Pack"
  • Test before releasing: Import your own pack and test all animations
  • Write a changelog: Include release notes in your GitHub releases
  • Optimize file sizes: Remove unnecessary frames to keep packs small
  • Be consistent: Use a consistent naming scheme for animation files
  • Include variety: Packs with 3-5 Pixlings are more appealing than single-Pixling packs
  • Credit others: If you used someone's artwork or animations, credit them in the description
🔧

Troubleshooting

"Update check failed" or "HTTP 404"

  • Verify your updateUrl is correct and publicly accessible
  • Make sure you uploaded pack.json as a release asset
  • Try opening the URL in a browser - you should see the JSON content

"No update available" when there should be

  • Check that the remote version is actually higher (e.g., 1.1.0 > 1.0.0)
  • Version comparison is case-sensitive for non-numeric parts
  • Try "Check for Updates" manually from Pack Details screen

Pack imports but animations don't work

  • Verify animation files are in the animations/ folder in the ZIP
  • Check that file names in pack.json match exactly (case-sensitive)
  • Make sure animations follow the proper JSON format (489 pixels per frame)
